# Telegram WebSocket Bridge Proxy (Go)

Прокси-сервер, который перенаправляет трафик Telegram Desktop через WebSocket-соединения к серверам Telegram, обходя блокировки прямых TCP-соединений к IP-адресам Telegram.

## Принцип работы

Telegram Desktop подключается к прокси как к обычному SOCKS5-серверу. Прокси перехватывает соединения к IP-адресам Telegram и перенаправляет их через WebSocket (`wss://kwsN.web.telegram.org/apiws`), который используется веб-версией Telegram. Остальной трафик (не к Telegram) проксируется напрямую без изменений.

Если WebSocket-соединение не удаётся установить, прокси автоматически переключается на прямое TCP-подключение (fallback).

## Исходный код

Оригинальная реализация на Python: [https://github.com/Flowseal/tg-ws-proxy](https://github.com/Flowseal/tg-ws-proxy)

Данная версия — полный порт на Go в один файл.

## Сборка

### Требования

- Go 1.21+ (стандартная библиотека, без внешних зависимостей)

### Компиляция

```bash
# Инициализация модуля (один раз)
go mod init tg-ws-proxy

# Сборка
go build -o tg-ws-proxy main.go
```

### Кросс-компиляция

```bash
# Для Linux (amd64)
GOOS=linux GOARCH=amd64 go build -o tg-ws-proxy-linux tgws.go

# Для Windows
GOOS=windows GOARCH=amd64 go build -o tg-ws-proxy.exe tgws.go

# Для macOS (Apple Silicon)
GOOS=darwin GOARCH=arm64 go build -o tg-ws-proxy-mac tgws.go

# Для роутеров (MIPS, ARM и т.д.)
GOOS=linux GOARCH=mipsle GOMIPS=softfloat go build -o tg-ws-proxy-mipsle main.go
GOOS=linux GOARCH=arm GOARM=7 go build -o tg-ws-proxy-arm7 main.go
```

## Запуск

### Базовый запуск (настройки по умолчанию)

```bash
./tg-ws-proxy
```

По умолчанию:
- Слушает на `127.0.0.1:1080`
- Настроены DC2 и DC4 на IP `149.154.167.220`

### С параметрами

```bash
./tg-ws-proxy \
  -port 1080 \
  -host 127.0.0.1 \
  -dc-ip 2:149.154.167.220 \
  -dc-ip 4:149.154.167.220 \
  -v
```

### Параметры командной строки

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `-port`  | Порт прослушивания | `1080` |
| `-host`  | Адрес прослушивания | `127.0.0.1` |
| `-dc-ip` | Целевой IP для дата-центра (формат `DC:IP`, можно указать несколько раз) | `2:149.154.167.220`, `4:149.154.167.220` |
| `-v`     | Подробное логирование (debug) | выключено |

### Примеры

```bash
# Слушать на всех интерфейсах (для доступа из локальной сети)
./tg-ws-proxy -host 0.0.0.0 -port 1080

# Только DC2
./tg-ws-proxy -dc-ip 2:149.154.167.220

# Все 5 дата-центров
./tg-ws-proxy \
  -dc-ip 1:149.154.175.50 \
  -dc-ip 2:149.154.167.220 \
  -dc-ip 3:149.154.175.100 \
  -dc-ip 4:149.154.167.91 \
  -dc-ip 5:91.108.56.100
```

## Настройка Telegram Desktop

1. Откройте **Telegram Desktop**
2. Перейдите в **Настройки** → **Продвинутые настройки** → **Тип соединения** → **Использовать прокси**
3. Нажмите **Добавить прокси**
4. Заполните:
   - **Тип:** SOCKS5
   - **Адрес:** `127.0.0.1` (или IP машины с прокси)
   - **Порт:** `1080` (или тот, что указали при запуске)
   - **Имя пользователя:** оставить пустым
   - **Пароль:** оставить пустым
5. Нажмите **Сохранить** и выберите добавленный прокси

### Важно

- В настройках Telegram Desktop **не включайте** «Использовать IPv6» — IPv6 не поддерживается
- Прокси работает **без логина и пароля**
- Поддерживается только протокол **SOCKS5**

## Настройка мобильных клиентов

### Android / iOS

Telegram на мобильных устройствах также поддерживает SOCKS5:

1. **Настройки** → **Данные и память** → **Прокси**
2. **Добавить прокси** → **SOCKS5**
3. Укажите адрес и порт прокси-сервера

> Если прокси запущен на компьютере в локальной сети, используйте
> его локальный IP (например, `192.168.1.100`) вместо `127.0.0.1`
> и запустите прокси с флагом `-host 0.0.0.0`.

## Как работает прокси (подробно)

```
Telegram Desktop
       │
       ▼
  SOCKS5 подключение
       │
       ▼
┌──────────────────────┐
│   tg-ws-proxy (Go)   │
│                      │
│  1. Принимает SOCKS5 │
│  2. Определяет IP    │
│     └─ Telegram IP?  │
│        ├─ Да:        │
│        │  Извлекает   │
│        │  DC из init  │
│        │  пакета      │
│        │  ↓           │
│        │  WebSocket   │
│        │  к kwsN.web. │
│        │  telegram.org│
│        │  (TLS+WS)   │
│        │              │
│        └─ Нет:        │
│           Прямой TCP  │
│           passthrough │
└──────────────────────┘
       │
       ▼
  Серверы Telegram
  (через WebSocket
   или прямой TCP)
```

### Механизмы надёжности

- **Пул WebSocket-соединений** — заранее устанавливает соединения для быстрого подключения
- **Автоматический TCP fallback** — если WebSocket не работает, переключается на прямое соединение
- **Чёрный список DC** — дата-центры, отвечающие 302 на все WS-запросы, автоматически переводятся на TCP
- **Cooldown** — при ошибках WS повторные попытки откладываются на 60 секунд
- **Разделение MTProto сообщений** — корректно разбивает пакеты для WS-relay

## Запуск как сервис (systemd)

Создайте файл `/etc/systemd/system/tg-ws-proxy.service`:

```ini
[Unit]
Description=Telegram WebSocket Bridge Proxy
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/tg-ws-proxy -host 127.0.0.1 -port 1080 -dc-ip 2:149.154.167.220 -dc-ip 4:149.154.167.220
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable tg-ws-proxy
sudo systemctl start tg-ws-proxy
sudo systemctl status tg-ws-proxy

# Просмотр логов
sudo journalctl -u tg-ws-proxy -f
```

## FAQ

### Зачем нужен этот прокси?

В некоторых сетях прямые TCP-подключения к IP-адресам Telegram заблокированы (DPI, файрвол). WebSocket-подключения через TLS (порт 443) часто остаются доступными, так как выглядят как обычный HTTPS-трафик.

### Почему Go, а не Python?

- Один статически скомпилированный бинарник без зависимостей
- Легко кросс-компилируется (роутеры, NAS, VPS)
- Меньше потребление памяти
- Не требует установки Python и pip

### Безопасно ли это?

Прокси не читает и не модифицирует ваш трафик Telegram (он зашифрован MTProto). Единственное, что делает прокси — оборачивает TCP-пакеты в WebSocket-фреймы.

### Работает ли с MTProxy?

Нет. Этот прокси — SOCKS5 и работает с обычным MTProto-протоколом Telegram Desktop. MTProxy — отдельный протокол.
